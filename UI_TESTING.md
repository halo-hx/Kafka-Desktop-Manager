# Kafka Desktop Manager 前端 UI 测试说明

> 技术栈：React 18、TypeScript、Zustand、`react-virtuoso`、`lucide-react`、Tauri `invoke()`  
> 状态：`connectionStore`、`uiStore`、`clusterStore`、`messageStore`、`settingsStore`  
> 本文档侧重**界面表现、交互与可访问性**；与 `TESTING.md` 功能用例配合使用。

---

## 优先级与编号

- **UI-{模块}-xxx**：与界面/组件强相关的用例 ID  
- 优先级：**P0** 阻断体验，**P1** 重要，**P2** 增强

---

## 一、App 布局（侧栏 + 标签 + 主面板）

### 1.1 组件渲染

| ID | 标题 | 前置 | 步骤 | 预期 | 优先级 |
|----|------|------|------|------|--------|
| UI-APP-001 | 根布局结构 | 应用启动 | 观察窗口：左侧侧栏、主区、（若有）顶部栏/标签栏 | 主要区域可见；无布局重叠或溢出（常见分辨率 1280×720） | P0 |
| UI-APP-002 | 侧栏图标与文案 | 同上 | 逐项核对侧栏项图标（lucide）、中文标签 | 图标与功能语义一致；无截断（或可 Tooltip） | P1 |
| UI-APP-003 | 连接状态指示 | 未连接 / 已连接各测一次 | 查看侧栏或顶栏连接指示 | 未连接与已连接视觉区分明确 | P0 |

### 1.2 交互

| ID | 标题 | 步骤 | 预期 | 优先级 |
|----|------|------|------|--------|
| UI-APP-010 | 侧栏切换主面板 | 点击各导航项 | 主内容切换；URL/内部路由与选中项一致 | P0 |
| UI-APP-011 | Tab 关闭与保留焦点 | 若有多 Tab：打开多个、关闭当前 | 关闭后焦点落在新 Tab 或主区；无死焦点 | P1 |

### 1.3 Zustand（uiStore）

| ID | 标题 | 验证点 | 优先级 |
|----|------|--------|--------|
| UI-APP-020 | 侧栏选中项与 store | 切换导航后 `uiStore`（或等价）记录当前模块 | 是否持久化符合设计 | P1 |
| UI-APP-021 | 面板折叠（若有） | 折叠侧栏后再展开 | 主区宽度变化正确、无闪动 | P2 |

### 1.4 错误与网络

| ID | 标题 | 步骤 | 预期 | 优先级 |

| UI-APP-030 | invoke 失败全局提示 | 断网或杀 Kafka 后触发需 IPC 的操作 | Toast/Modal 错误；可关闭；无白屏 | P0 |

### 1.5 可访问性与布局

| ID | 标题 | 步骤 | 预期 | 优先级 |
|----|------|------|------|--------|

## Store 与界面对照（快速参考）

| Store | 主要 UI 关联 |
|--------|----------------|
| `connectionStore` | 连接列表、对话框、连接状态 |
| `uiStore` | 侧栏、主题类即时 UI、布局状态 |
| `clusterStore` | 集群概览、Broker、Topic 列表元数据 |
| `messageStore` | 消息查看器、发送面板、导出状态 |
| `settingsStore` | 设置页各 Tab、编辑器/消息默认值 |
